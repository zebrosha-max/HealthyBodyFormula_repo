# SDD PHASE 1: PRD & USER STORIES

## 1.1 Product Requirements Document (PRD)

**Название продукта:** Healthy Body Formula (HBF) Web App v2.0
**Цель:** Преобразовать портфолио в сервисную платформу для монетизации контента и предоставления услуг нутрициолога.

**Ключевые возможности:**
1.  **Идентификация:** Автоматическая регистрация/вход через Telegram User ID.
2.  **Услуги:** Витрина консультаций с кнопками записи.
3.  **Профиль:** Личный кабинет со статусом подписки.
4.  **Умные Рецепты:** Расширенная фильтрация (время, ккал, тип) и избранное.
5.  **Платный контент:** Система "Freemium" для гайдов (Blur-эффект для неоплаченного контента).

## 1.2 User Stories (Пользовательские истории)

| ID | Роль | Я хочу... | Чтобы... | Критерии приемки |
| :--- | :--- | :--- | :--- | :--- |
| **US-01** | User | Видеть список услуг нутрициолога | Понять, что я могу купить и сколько это стоит | Отдельный экран `#services`. Список карточек с ценой и кнопкой "Записаться" (ведет в ЛС). |
| **US-02** | User | Фильтровать рецепты по времени, ккал и типу | Быстро найти подходящее блюдо | Система из 4 выпадающих списков (Select). Фильтрация происходит по пересечению условий (AND logic). |
| **US-03** | User | Добавлять рецепты в "Избранное" | Не терять понравившиеся блюда | Иконка сердца на карточке. Состояние сохраняется между сессиями. |
| **US-04** | User | Видеть список гайдов, но читать платные только после оплаты | Оценить полезность перед покупкой | Платные гайды в общем списке. При открытии без подписки — заблюренный текст и кнопка "Купить". |
| **US-05** | User | Иметь личный профиль | Видеть свой статус и управлять подпиской | Экран `#profile`. Отображение аватарки TG, имени, статуса (Free/Pro). |
| **US-06** | Admin | Автоматически идентифицировать пользователя | Не заставлять его вводить логин/пароль | Использование `Telegram.WebApp.initData` для тихой аутентификации. |
