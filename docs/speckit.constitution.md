# SDD PHASE 0: PROJECT CONSTITUTION

### 1. Основные Принципы
*   **Mobile-First TWA:** Интерфейс проектируется строго под мобильные устройства в контексте Telegram Web App. Учитываем безопасные зоны (safe areas), нативные цвета темы и тактильный отклик (Haptic Feedback).
*   **Security First:** Логика проверки прав доступа ("платный/бесплатный") должна выполняться на стороне сервера (или BaaS). Клиентская защита (CSS blur) — только декорация; данные платного контента не должны загружаться на клиент без валидации.
*   **Zero-Dependency (Frontend):** Продолжаем придерживаться Vanilla JS (ES6+) там, где это возможно, чтобы избежать переусложнения сборки.
*   **State Management:** Вводим явное управление состоянием пользователя (Guest vs Subscriber) и фильтрами.

### 2. Стандарты Кода
*   **HTML:** Семантическая верстка. Использование `data-attributes` для фильтрации и роутинга.
*   **CSS:** Использование CSS Variables для темизации. BEM-подобный нейминг классов.
*   **JS:**
    *   Модульная структура внутри единого файла (или разделение на ES Modules в будущем).
    *   Асинхронные функции (`async/await`) для работы с Supabase.
    *   Обработка ошибок при работе с сетью.
